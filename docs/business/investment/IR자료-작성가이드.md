## IR 자료란?

**IR(Investor Relations)** = 투자자와 관계를 맺는 활동

투자자를 대상으로 기업의 각종 정보를 제공하는 행위이며, **IR 자료(IR Material)**는 스타트업이 투자유치 과정에서 활용하는 문서 일체를 말합니다.

---

## 2025-2026년 투자 트렌드

### 주목받는 분야

| 분야 | 트렌드 | CareOn 관련성 |
|------|--------|--------------|
| **AI/LLM** | 생성형 AI, 특화 AI 솔루션 | **핵심** - AI 에이전트 |
| **소상공인 SaaS** | 디지털 전환, 운영 자동화 | **핵심** - 디지털 트윈 OS |
| **핀테크** | 결제, 데이터 기반 금융 | **높음** - VAN 대리점 |
| **버티컬 SaaS** | 산업 특화 솔루션 | **높음** - 소상공인 특화 |

### 투자자가 주목하는 포인트 (2025년 기준)

1. **AI 활용의 실질적 가치**: 단순 AI 도입이 아닌 실제 비용 절감/매출 증대 효과
2. **데이터 자산**: 경쟁사가 쉽게 복제할 수 없는 독점 데이터
3. **BM의 명확성**: 어떻게 돈을 벌 것인가
4. **팀의 실행력**: 해당 분야 경험과 전문성

## IR 자료 필수 7가지 항목

### 1. 문제 정의와 해결책

> "투자자는 스타트업이 아니라 **문제를 해결하려는 시도**에 투자한다"

- 왜 **지금** 해결해야 하는가?
- 해결 과정에서 얼마나 **기회**가 생기는가?
- 기존 해결책의 한계는 무엇인가?

### 2. 솔루션 설명

- 기술 중심 X → **고객 가치 중심** O
- 고객이 얼마나/어떻게 편해지는가?
- 소비자가 돈을 낼 니즈가 있는가?

**작성 팁**: 기술 자랑보다 "고객이 겪는 변화"를 설명

### 3. 시장 규모 (TAM, SAM, SOM)

```
TAM (Total Addressable Market) - 전체 시장
 └── SAM (Serviceable Addressable Market) - 유효 시장
      └── SOM (Serviceable Obtainable Market) - 수익 시장
```

- 단순히 "시장이 크다" X
- **SOM**: 지금 당장 공략 가능한 시장이 얼마인가?
- **어떻게** 그 시장을 뚫을 수 있는가?

**주의**: 시장 규모를 "산술적으로만" 계산하지 말 것 (출처와 로직 중요)

### 4. 경쟁사 분석

- "경쟁사가 없습니다" = 시장 조사 안 했다는 신호
- 경쟁자 있음 = 시장이 검증됨
- 경쟁사 대비 **차별점** 명확히

**작성 팁**: 경쟁사를 깎아내리지 말고, 우리의 차별점을 강조

### 5. 비즈니스 모델/수익 모델

- 어떻게 돈을 벌 것인가?
- 지속 가능한가?
- 확장 가능한가?

**핵심 질문**:
- 유닛 이코노믹스 (CAC, LTV, Payback Period)
- 마진 구조
- 스케일업 시 변동 비용

### 6. 팀 구성

> 팀 구성이 없는 IR자료 = 읽지 않음  
> 허위 기재 = 투자 철회

- **정직하게** 모든 구성원 이력 + 실제 사진
- 이 문제를 왜 우리 팀이 풀 수 있는가?
- 부족한 부분은 어떻게 채울 계획인가?

### 7. 재무 계획

- 투자금을 **어떻게** 쓸 것인가?
- 투입 비용, 예상 손익계산서
- 손익분기점(BEP) 시점
- 예상 영업이익

**중요**: 투자금 사용처가 명확해야 함 (인력 채용 X%, 마케팅 X% 등)

## IR 자료 작성 팁

### 세콰이아 캐피탈 사업계획 목차 (참고용)

```
1. Company Purpose (한 문장 요약)
2. Problem (문제)
3. Solution (해결책)
4. Why Now (왜 지금인가)
5. Market Size (시장 규모)
6. Competition (경쟁)
7. Product (제품)
8. Business Model (수익 모델)
9. Team (팀)
10. Financials (재무)
```

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **간결함** | 바쁜 투자자/심사역 대상, 10-15장 권장 |
| **일관성** | 문제-솔루션-시장-수익모델이 하나의 흐름 |
| **실현 가능성** | 실제 실행 가능한 수치와 전략 |
| **시각화** | 숫자보다 그래프, 텍스트보다 다이어그램 |

### 자주 하는 실수

- ❌ 시장 규모를 부풀리기
- ❌ 경쟁사를 무시하거나 폄하
- ❌ 기술 자랑만 하고 고객 가치 미설명
- ❌ 팀 소개 생략
- ❌ 투자금 사용처 불명확

## CareOn IR 자료 작성 시 강조점

### 1. 문제 정의

**소상공인이 겪는 핵심 문제**:

| 문제 | 현실 |
|------|------|
| **운영 복잡성** | POS, 마케팅, 회계, 노무 - 각각 다른 시스템 |
| **전문가 비용** | 세무사, 노무사, 마케터 위임 비용 부담 |
| **데이터 사일로** | 각 시스템 데이터가 분리되어 통합 분석 불가 |
| **디지털 격차** | 디지털 도구 사용 역량 부족 |

**Why Now?**:
- AI 기술 발전으로 자동화 비용 급감
- 정부의 소상공인 디지털 전환 정책 확대
- 소상공인 고령화로 운영 부담 증가

### 2. 솔루션

**CareOn 디지털 트윈 OS**:

```
┌─────────────────────────────────────────────────┐
│            CareOn Digital Twin OS               │
├─────────────────────────────────────────────────┤
│  POS + 결제  │  매출 분석  │  AI 에이전트      │
│  (Phase 1)   │  (Phase 3) │  (Phase 5)        │
├─────────────────────────────────────────────────┤
│           통합 데이터 레이어 (온톨로지)          │
├─────────────────────────────────────────────────┤
│     VAN 결제 데이터  │  음성 데이터 (Phase 4)  │
└─────────────────────────────────────────────────┘
```

**고객 가치**:
- 하나의 시스템으로 매장 운영 전체 관리
- AI가 마케팅/회계/노무 업무 자동 처리
- 데이터 기반 의사결정 지원

### 3. 시장 규모

| 시장 | 규모 | 근거 |
|------|------|------|
| **TAM** | 국내 소상공인 ~700만 사업자 | 소상공인기본법 통계 |
| **SAM** | POS 사용 소상공인 ~200만 | 유통업, 음식업 중심 |
| **SOM** | 초기 타겟 ~10,000 매장 | 스마트상점 연계, 수도권 |

**관련 시장**:
- POS/결제 시장
- 소상공인 SaaS 시장
- AI 자동화 시장

### 4. 차별점 (경쟁 우위)

| 차별점 | 설명 | 경쟁사 대비 |
|--------|------|------------|
| **VAN 대리점 지위** | 결제 데이터 직접 접근 | 토스/네이버 등은 PG만 가능 |
| **음성 데이터 수집** | 매장 내 음성 → 독점 데이터셋 | 경쟁사 없음 |
| **온톨로지 기반** | 팔란티어 방식 데이터 모델 | 소상공인 특화 |
| **AI 에이전트** | 업무 자동화 + 의사결정 지원 | 단순 대시보드와 차별화 |

### 5. 비즈니스 모델

**수익 구조**:

| 수익원 | 모델 | 예상 단가 |
|--------|------|----------|
| **VAN 수수료** | 거래당 수수료 | 거래액의 0.X% |
| **SaaS 구독료** | 월정액 | 월 X만원 |
| **AI 에이전트** | 기능별 과금 | 기능당 월 X만원 |
| **데이터 분석** | 프리미엄 분석 | 건당/월정액 |

### 6. 현재 진행 상황

| Phase | 내용 | 상태 |
|-------|------|------|
| Phase 1 | 나이스페이 결제 파이프라인 | **완료** |
| Phase 2 | 메뉴/상품 관리 | 예정 |
| Phase 3 | 디지털 트윈 대시보드 | 예정 |
| Phase 4 | 음성 수집 인터페이스 | 예정 |
| Phase 5 | AI 분석 엔진 | 예정 |

**MVP 검증 완료** (2025.11.30):
- 서버 ↔ 프론트엔드 연동 완료
- 나이스페이 결제창 호출 성공
- Webhook 수신 파이프라인 구축

### 7. 투자금 사용 계획 (예시)

| 항목 | 비율 | 용도 |
|------|------|------|
| 개발 인력 | 50% | 풀스택 개발자, AI 엔지니어 |
| 마케팅 | 20% | 초기 고객 확보, B2B 영업 |
| 인프라 | 15% | 서버, 데이터 스토리지 |
| 운영 | 15% | 법무, 회계, 기타 운영비 |
