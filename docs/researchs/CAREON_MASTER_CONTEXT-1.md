## 2.2 수익 모델 (3-Layer Structure)

```
Layer 1: VAN 수수료
├─ 고객의 모든 카드 결제가 나이스페이 경유
├─ 결제 건당 수수료의 일부 귀속
└─ 예상: 월 5,000원/점포

Layer 2: AI 구독료
├─ 베이직: 월 19,900원
├─ 프로: 월 39,900원
└─ 엔터프라이즈: 월 99,900원

Layer 3: 데이터 수익 (장기)
├─ 트렌드 리포트 → 제조사에 판매
├─ 상권 분석 → 프랜차이즈에 판매
└─ 소비 패턴 → 마케팅 회사에 판매
```

## 2.3 핵심 무기: 토스프론트 + 케어온 AI 번들

**번들 전략의 핵심**:
1. 토스프론트 = 무료 (VAN 대리점 지원, 실제 부담 3만원)
2. 케어온 AI = 월 19,900원
3. **고객 입장**: "무료로 POS 받고 AI 서비스까지"

**중요한 구조적 이점**:
```
토스 = 프론트엔드 (UI/UX, 결제 앱)
CareOn = 백엔드 (데이터, AI, VAN 연동)

결제 데이터 흐름:
고객 카드 → 토스플레이스 앱 → 나이스페이 VAN → CareOn 대시보드

VAN 대리점으로서 접근 가능한 데이터:
```

## 3.1 팔란티어 방식의 온톨로지 적용

```
┌────────────────────────────────────────────────────────────┐
│                    CareOn OS 아키텍처                      │
├────────────────────────────────────────────────────────────┤
│ [엣지 레이어]                                              │
│   POS + CCTV + 키오스크 + 출입통제 + 조명/음악             │
│                          ↓                                 │
│ [통합 레이어]                                              │
│   온톨로지 엔진 (모든 데이터를 "매장 언어"로 번역)          │
│                          ↓                                 │
│ [AI 레이어]                                                │
│   LLM API (GPT/Claude) - 상황 해석 및 의사결정 지원        │
│                          ↓                                 │
│ [UI 레이어]                                                │
│   점주 앱 + 본사 대시보드 (토스 압도하는 직관성)           │

```

```python
        # 1단계: 도메인 지식 여부 판별

            return self.local_ai.respond(query)
        else:
            # GPT/Claude API 호출 (느림, 비용 발생)
            return self.cloud_api.respond(query)
    
    def is_domain_knowledge(self, query):
        """
        도메인 지식 예시:
        - "칠성사이더 가격?" → 로컬
        - "런치 세트 구성?" → 로컬
        - "2026년 세금 신고 방법?" → 클라우드
```

- 컴퓨팅 비용 절감 (로컬 90% → API 10%만)

2. **온톨로지 설계**: 팔란티어 방식 적용
3. **엣지 컴퓨팅**: 고객 매장 전기세로 컴퓨팅 비용 전가
4. **API 경제성**: 도메인 지식은 로컬, 일반 지식은 클라우드

## 10.3 영업 원칙

2. **FOMO 전술**: 경쟁사 언급으로 긴급성 부여
3. **스토리텔링**: 제품이 아닌 성공 스토리 판매
4. **반박 제거**: 예상 질문에 선제 답변

# 부록: 용어 정의

## A. VAN (Value Added Network)
- **정의**: 카드 결제 중개 네트워크
- **비유**: 카드사와 가맹점 사이의 우체국
- **메커니즘**: 고객 카드 긁음 → VAN사 중계 → 카드사 승인 → POS 승인

## B. 파인튜닝 (Fine-tuning)
- **정의**: 사전 학습된 AI에 도메인 지식 추가 학습
- **비유**: 일반인 → 해군 운전병 변환
- **메커니즘**: 5천개 Q&A 데이터 → 3시간 학습 → 맞춤 AI 완성

## C. 온톨로지 (Ontology)
- **정의**: 데이터 간 관계를 정의하는 체계
- **비유**: 사전 + 백과사전 + 관계도의 조합
- **메커니즘**: 객체(노드) + 관계(엣지) + 속성(프로퍼티)

## D. 엣지 컴퓨팅 (Edge Computing)
- **정의**: 데이터가 생성되는 현장에서 처리
- **비유**: 본사 서버 대신 매장 내 컴퓨터에서 처리
- **장점**: 속도↑, 보안↑, 클라우드 비용↓

## E. PG (Payment Gateway)
- **정의**: 온라인 결제 중개 서비스
- **비유**: 오프라인 VAN의 온라인 버전
- **대표사**: NHN KCP, 이니시스, 토스페이먼츠
